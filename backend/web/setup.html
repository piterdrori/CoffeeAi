Setup Guide

Install the app and connect your phone

Home Setup Admin

Method A — USB + Android Studio

  1. Install Android StudioDownload from developer.android.com. Include Android SDK 34 and JDK 17.
  2. Enable USB debuggingOn phone: Settings → About → tap Build number 7 times → Developer options → USB debugging ON.
  3. Open the projectIn Android Studio: Open folder android/ from this repo.
  4. Connect phone via USBAccept the debugging prompt on your phone.
  5. Click RunGreen play button in Android Studio. App installs directly — no APK file needed.

Method B — WiFi download

  1. Build & publish APK on PCRun scripts\publish-apk.ps1 from the project root (requires JDK + Android SDK).
  2. Start backenddocker compose up -d or uvicorn main:app --port 8080 in backend/.
  3. Find your PC IPRun ipconfig in PowerShell. Use the IPv4 address (e.g. 192.168.1.42).
  4. Download on phoneSame WiFi → open http://YOUR_PC_IP:8080 → Download APK.
  5. Allow unknown appsWhen prompted, allow your browser to install apps.
  6. Open app → SettingsSet Local backend URL to http://YOUR_PC_IP:8080 and API key from backend/.env.

Method C — Android emulator (PC testing)

  1. Start emulatorAndroid Studio → Device Manager → run a virtual device.
  2. Run the appOpen android/ in Android Studio → green Run button.
  3. Backend URL in appUse http://10.0.2.2:8080 (not localhost, not your Wi‑Fi IP).
  4. API keySame bootstrap value as API_KEY in backend/.env (or Vercel for cloud). Enter it in the app Settings screen — it is never shown by the server.

First-run checklist

  1. Set API key in app SettingsMust match API_KEY in backend .env.
  2. Add Hugging Face tokenModels tab → required for Gemma 3 1B (gated). Accept license at huggingface.co first.
  3. Download a modelGemma 3 1B (~620 MB) for text. Gemma 4 E2B (~2.6 GB) for images.
  4. Configure personalityOn PC open /admin → set system prompt and upload knowledge files.
  5. Sync memoryIn app Settings → Sync memory now, or chat online to prefetch automatically.

Windows firewall

If your phone cannot reach the backend, allow port 8080:

netsh advfirewall firewall add rule name="Personal Edge AI" dir=in action=allow protocol=TCP localport=8080
